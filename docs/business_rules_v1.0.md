## 1. システム概要

DPCデータ（HファイルおよびEFファイル）を取り込み、看護必要度を日次で集計・分析することで、入院基本料・加算の施設基準充足状況を確認するツール。

---

## 2. DPCデータの概要

### Hファイルとは

**看護師が手動で評価した看護必要度の結果を記録するファイル。**

- 患者1名×実施日1日×ペイロード種別1つ = 1行
- B項目（患者の状態＋介助の実施）は常にHファイルから取得
- A項目は看護必要度Ⅰの場合のみHファイルから取得（Ⅱでは使用しない）
- 看護必要度の判定対象フラグ（TAR0010）もHファイルに含まれる

### EFファイルとは

**入院中の患者に対して行われた全ての診療行為の明細データ（レセプト電算コード付き）。**

- 患者1名×実施日1日×行為1つ = 1行（数万〜数十万行の巨大ファイル）
- A項目（看護必要度Ⅱの場合）・C項目の判定に使用
- レセプト電算コード（EF-9）をマスタと照合してA/C項目の該当有無を判定する

### データソースの使い分け

| 評価方式 | A項目 | B項目 | C項目 |
|---------|-------|-------|-------|
| 必要度**Ⅰ** | Hファイル（手動評価）＋EFファイル（一部） | Hファイル | EFファイル |
| 必要度**Ⅱ** | **EFファイル（全て自動判定）** | Hファイル | EFファイル |

> B項目は常にHファイル、C項目は常にEFファイルから取得。  
> A項目のソースのみ評価方式によって異なる。

### 統合キー（H×EFの突合）

| キー項目 | Hファイル | EFファイル |
|---|---|---|
| 患者識別 | データ識別番号 | EF-2 データ識別番号 |
| 実施日 | 実施年月日（yyyymmdd） | EF-24 実施年月日 |
| 病棟 | 病棟コード | EF-28 病棟コード |

## 4. 処理の全体フロー

> **詳細な実装パイプラインは `calculation_logic_v0.1.md` のセクション6を参照。**

```
【入力】
  Hファイル（タブ区切りテキスト）＋ EFファイル（タブ区切りテキスト）
        ↓
【Step 1: パース】
  parseHFile   → HRecordEntry[]（全列を文字列配列で保持）
  parseEfFile  → EfActEntry[]（9列を名前付きフィールドで保持）
  ※ EFファイルは外来データ・不要データ区分を除外
        ↓
【Step 2: 空の評価マップを作る】
  H/EFファイルの患者×日付から GenIIDailyScore の Map を生成
  全フィールドを 0 / false で初期化
        ↓
【Step 3: B項目を埋める】
  HRecordEntry（ASS0021）からペイロード1〜11を抽出
  → b1〜b7, b2_assist〜b5_assist を埋める
        ↓
【Step 4: A項目を埋める】
  EfActEntry をレセプト電算コードマスタと照合
  → a1〜a7, a6_1〜a6_11 を埋める
        ↓
【Step 5: C項目を埋める】
  EfActEntry をレセプト電算コードマスタと照合
  → 有効期間を展開して c15〜c23 を埋める
        ↓
【Step 6: A3・A7の有効期間展開】
  A3（最大7日間）・A7（2日間）のスコアを展開
        ↓
【Step 7: 施設基準等を満たす該当患者の割合判定】
  判定パターン（P1〜P3）に照らして meetsP1〜P3 を算出
        ↓
【Step 8: 結果出力】
  看護必要度詳細テーブル（レコード詳細画面 Tab 3）
  Excel出力（テーブルと同じ列構成・表示名）
```

### 5-4. レセプト電算コードマスタ

**参照ファイル：** `general_ward_ac_item_code_master_v0.2.tsv`  
**件数：** 2,591件（A項目：1,059件 / C項目：1,532件）  
**根拠：** 厚労省通知別表（「重症度、医療・看護必要度」に係る評価票 評価の手引き）

| フィールド | 型 | 内容 |
|---|---|---|
| `category` | STRING | `A` または `C` |
| `item_no` | STRING | 項目番号（例：`1`、`6`、`15`） |
| `sub_item` | STRING | 子項目記号（例：`①`〜`⑪`。なしは空欄） |
| `item_name` | STRING | 項目名称 |
| `receipt_code` | STRING(9) | レセプト電算コード（9桁） |
| `procedure_name` | STRING | 診療行為名称 |
| `level1_only` | INTEGER | 下記参照 |

#### `level1_only` フラグの意味

> ⚠️ **このフラグ名は誤解を招きやすい。正確な意味は以下の通り。**

| 値 | 意味 |
|:--:|------|
| `0` | Ⅰ・Ⅱ共通で使用するレセプトコード |
| `1` | **Ⅰにおいて Hファイル（ASS0013）からも取得する項目** のレセプトコード。Ⅱでは **EFファイルから評価する** ため、このフラグが `1` でも **Ⅱでの評価対象外ではない** |

つまり、Ⅱの判定時に `level1_only = 1` のコードを除外してはいけない。

**判定処理での使い分け：**

```
一般病棟Ⅱの場合：
  A項目 → level1_only = 0 および 1 の両方を使用（全てEFから判定）
  ただし、ⅠにおいてHファイルから取得する項目（⑤⑩⑪）も
  ⅡではEFファイルのレセプトコードで判定する

一般病棟Ⅰの場合：
  level1_only = 1 の項目 → Hファイル（ASS0013）から取得
  level1_only = 0 の項目 → EFファイルから取得
```

### 6-2. B項目（患者の状態＋介助の実施）

**データソース：Hファイル ASS0021（Ⅰ・Ⅱ共通）**

#### 患者の状態（ペイロード1〜7）

| Payload | 項目名 | スコア |
|:-------:|--------|:------:|
| 1 | 寝返り | 0:できる / 1:何かにつかまればできる / 2:できない |
| 2 | 移乗（患者の状態） | 0:自立 / 1:一部介助 / 2:全介助 |
| 3 | 口腔清潔（患者の状態） | 0:自立 / 1:要介助 |
| 4 | 食事摂取（患者の状態） | 0:自立 / 1:一部介助 / 2:全介助 |
| 5 | 衣服の着脱（患者の状態） | 0:自立 / 1:一部介助 / 2:全介助 |
| 6 | 診療・療養上の指示が通じる | 0:はい / 1:いいえ |
| 7 | 危険行動 | 0:ない / 1:ある |

#### 介助の実施（ペイロード8〜11）

| Payload | 対応する患者の状態 | 値 |
|:-------:|:------------------:|:--:|
| 8 | 移乗（payload 2） | 0:実施なし / 1:実施あり |
| 9 | 口腔清潔（payload 3） | 0:実施なし / 1:実施あり |
| 10 | 食事摂取（payload 4） | 0:実施なし / 1:実施あり |
| 11 | 衣服の着脱（payload 5） | 0:実施なし / 1:実施あり |

> B1（寝返り）・B6（指示が通じる）・B7（危険行動）には介助の実施フィールドがない。

#### B項目の得点算出ルール

B2〜B5 は **「患者の状態」×「介助の実施」** の掛け算構造になっている。

**基本ルール：介助の実施が `0`（なし）の場合、患者の状態がどうであっても得点は `0` になる。**

```
B項目得点 = 患者の状態スコア × 介助の実施フラグ

例：
  移乗: 患者の状態=2（全介助）, 介助の実施=1（実施あり） → 得点 = 2
  移乗: 患者の状態=2（全介助）, 介助の実施=0（実施なし） → 得点 = 0
  寝返り: 患者の状態=2 → 得点 = 2（介助フィールドなし、そのまま）
  指示が通じる: 患者の状態=1 → 得点 = 1（介助フィールドなし、そのまま）
```

> 🔲 **上記は基本ルールの概要。B2〜B5の正確な算出マトリクスは要確定。**  
> 具体的には「患者の状態=1（一部介助）、介助の実施=0」のケースの得点が確定していない。

**bTotal = B1〜B7 の得点合計**（介助の掛け算適用後）

## 8. 判定対象フラグ（TAR0010）

TAR0010 は Hファイルのペイロードとして記録される。集計対象の判断に使用。

| 値 | 内容 | 集計への影響 |
|:--:|------|-------------|
| 0 | 判定対象 | **集計対象** |
| 1 | 短期滞在手術等基本料算定症例 | **一般病棟のみ集計対象**（0扱い）、それ以外は除外 |
| 2 | 15歳未満 | 除外 |
| 3 | 産科 | 除外 |
| 4 | 外泊日 | 除外 |
| 5 | 退院日（入院日＝退院日は除く） | 除外 |

**優先順位：** 複数条件該当時は値の小さいものを優先。  
**例外：** 一般病棟で15歳未満の短期滞在手術患者 → 「1」ではなく「2」。

## 9. 施設施設基準等を満たす該当患者の割合判定ルール

### 9-1. 判定の基本構造

1. TAR0010 = 0（一般病棟は 0 or 1）の患者・日のみ集計母数
2. 各患者・日が基準を満たすか判定
3. 割合（%）を算出し、入院料ごとの閾値と比較
4. 入院基本料と加算の施設基準は**独立して判定**

### 9-2. 判定パターン一覧

| パターンID | 条件式 | 主な該当入院料 |
|:----------:|--------|---------------|
| **P1** | A≥3 **または** C≥1 | 急性期一般1（基準①） |
| **P2** | (A≥2 **かつ** B≥3) **または** A≥3 **または** C≥1 | 急性期一般2〜5、10対1系 |
