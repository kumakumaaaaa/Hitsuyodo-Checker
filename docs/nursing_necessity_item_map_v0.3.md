# 看護必要度 評価項目 × 評価票 対応マップ

**バージョン:** v0.3  
**作成日:** 2026-02-21  
**ステータス:** 確認済み（項目・評価票対応・選択肢・評価構造）  
**変更内容（v0.2→v0.3）:** A項目番号の位置づけに関する注記を追記

---

## 概要

看護必要度の評価はA・B・Cの3カテゴリで構成される。  
評価票の種別によって評価対象となる項目が異なるため、システム実装時は評価票種別との組み合わせで項目を管理する必要がある。

### 評価票の種別

| 評価票コード | 名称 |
|---|---|
| `GENERAL_I` | 一般病棟用（必要度Ⅰ） |
| `GENERAL_II` | 一般病棟用（必要度Ⅱ） |
| `ICU` | ICU用（必要度Ⅱ） |
| `HCU` | HCU用（必要度Ⅰ・Ⅱ） |
| `REHABILITATION` | リハビリテーション用 |

> **⚠️ 項目番号の位置づけについて**  
> このドキュメントの `#` 列の番号は、全評価票（一般・ICU・HCU・リハ）の全項目を横断的に整理するための**便宜的な通し番号**である。  
> 実装上の正式な項目番号（`item_no`）は `general_ward_ac_item_code_master` に記載の番号（一般病棟評価票上の番号）が正しく、このドキュメントの番号とは一致しない。**実装コードではこのドキュメントの番号を使用しないこと。**

## B 患者の状況等

### 基本ルール

B項目の評価構造は3パターンに分類される。

| パターン | 該当項目 | 構造 |
|---|---|---|
| **2択** | B1・B12・B13 | 単一選択肢で直接評価 |
| **3択** | B2・B3・B4・B5・B7・B11 | 単一選択肢（3値）で直接評価 |
| **親子3層構造** | B6・B8・B9・B10 | 「患者の状態」＋「介助の実施」→「患者の状況等」を自動算出 |

### 病棟種別による評価方式の違い（B6・B8・B9・B10）

| 病棟種別 | 評価方式 |
|---|---|
| 一般病棟・ICU・HCU | 「患者の状態」と「介助の実施」を個別入力 → 「患者の状況等」を自動算出 |
| リハ病棟 | 「患者の状況等」を直接入力（個別評価） |

### B6・B8・B9・B10 親子3層構造の選択肢

| 層 | フィールド名 | 選択肢 |
|---|---|---|
| 子①（入力） | 患者の状態 | 自立／一部介助／全介助 |
| 子②（入力） | 介助の実施 | 実施なし／実施あり |
| 親（自動算出） | 患者の状況等 | 介助なし／一部介助／全介助 |

> 一般病棟・ICU・HCUでは子①②を入力し親を自動算出。リハでは親を直接入力。
