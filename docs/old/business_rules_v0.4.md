## 1. システム概要

DPCデータ（HファイルおよびEFファイル）を取り込み、看護必要度を日次で集計・分析することで、入院基本料・加算の施設基準充足状況を確認するツール。

### 処理の全体フロー

```
【入力】
  Hファイル（タブ区切りテキスト）
  EFファイル（タブ区切りテキスト）
        ↓
【取込・パース】
  ヘッダ部の解析（施設コード・病棟コード・患者識別・日付）
  ペイロード部の解析（コード・バージョンで項目を特定）
        ↓
【統合処理】
  Hファイル（手動評価項目）× EFファイル（レセプト電算コード）を
  患者ID × 実施日 をキーに突合・統合
        ↓
【評価票種別の決定】
  病棟コード・入院料種別から使用する評価票を特定
        ↓
【スコア算出】
  評価票種別に応じたA・B・Cスコアを計算
  判定対象フラグ（TAR0010）により除外患者を除く
        ↓
【施設基準充足判定】
  入院基本料・加算ごとの閾値と比較して充足状況を判定
        ↓
【出力】
  集計レポート・充足状況ダッシュボード
```

## 2. データソース設計

### 基本原則

**「Hファイルには手動評価が必要な項目のみ記録される」**

看護必要度の評価方式（ⅠまたはⅡ）によって、各項目のデータ取得元が異なる。

| 評価方式 | データ取得元 | 対象項目 |
|---|---|---|
| 必要度**Ⅰ** | Hファイル（看護師による手動評価） | レセプト電算コードとの対応が困難な項目 |
| 必要度**Ⅱ** | EFファイル（レセプト電算コードから自動判定） | レセプト電算コードで代替可能な項目 |

### 評価票種別 × データソース対応表

| 評価票種別 | A項目ソース | B項目ソース | C項目ソース |
|---|---|---|---|
| 一般病棟Ⅰ（GENERAL_I） | Hファイル（ASS0013）＋EFファイル（一部） | Hファイル（ASS0021） | EFファイル |
| 一般病棟Ⅱ（GENERAL_II） | EFファイル | Hファイル（ASS0021） | EFファイル |
| ICU（ICU） | EFファイル | Hファイル（ASS0051） | 対象外 |
| HCU（HCU） | Hファイル（ASS0062）＋EFファイル（一部） | Hファイル（ASS0071） | 対象外 |
| リハ（REHABILITATION） | 🔲 要確認 | 🔲 要確認 | 対象外 |

> **C項目は評価票種別に関わらず常にEFファイルから取得する。**

### 一般病棟Ⅰ A項目のソース詳細

一般病棟Ⅰは「手動評価＋EFファイル」の混在構成になる。

| 子項目 | 項目名 | ソース |
|---|---|---|
| A12-① | 抗悪性腫瘍剤の使用（注射剤のみ） | EFファイル |
| A12-② | 抗悪性腫瘍剤の内服の管理 | EFファイル |

| A12-⑥ | 免疫抑制剤の管理（注射剤のみ） | EFファイル |
| A12-⑦ | 昇圧剤の使用（注射剤のみ） | EFファイル |
| A12-⑧ | 抗不整脈剤の使用（注射剤のみ） | EFファイル |
| A12-⑨ | 抗血栓塞栓薬の持続点滴の使用 | EFファイル |
| A12-⑩ | ドレナージの管理 | Hファイル（ASS0013 ペイロード7） |

### 統合キー

HファイルとEFファイルは以下のキーで突合する。

| キー項目 | Hファイル | EFファイル | 備考 |
|---|---|---|---|
| 患者識別 | データ識別番号 | データ識別番号 | 様式1と同一 |
| 実施日 | 実施年月日（yyyymmdd） | 実施年月日（yyyymmdd） | 日次で突合 |
| 病棟 | 病棟コード | 病棟コード | 実施日ごとにセット |

## 3. Hファイル取込ルール

### 3-1. ファイル仕様

| 項目 | 仕様 |
|---|---|
| 形式 | タブ区切りテキスト（TSV） |
| 文字列 | 先頭が"0"の場合も文字列として保持すること（数値変換禁止） |
| 空欄 | 情報なしの場合はNull（"0"やスペース埋めなし） |

### 3-2. ヘッダ部の解析

| フィールド | 桁数・形式 | 備考 |
|---|---|---|
| 施設コード | 9桁（都道府県2桁＋医療機関コード7桁） | 区切りなし |
| 病棟コード | 10桁以下・左詰め | 病院独自コード。一般/一般以外の区別が可能なこと |
| データ識別番号 | — | 複数回入退院でも同一コード。様式1と一致 |
| 退院年月日 | 8桁（yyyymmdd） | 入院中の場合は`00000000` |
| 入院年月日 | 8桁（yyyymmdd） | — |
| 実施年月日 | 8桁（yyyymmdd） | 日次でセット |

### 3-3. ペイロード種別（コード）一覧

| コード | 評価票種別 | バージョン | 作成対象 |

### 3-4. ASS0013 ペイロード構造（一般病棟A項目・必要度Ⅰ）

| ペイロード番号 | 項目名 | 値 | 備考 |
|---|---|---|---|
| 1 | 創傷処置（褥瘡の処置を除く） | 0:なし / 1:あり | — |
| 2 | 呼吸ケア（喀痰吸引のみの場合を除く） | 0:なし / 1:あり | — |
| 3 | （空欄） | Null | — |
| 4 | （空欄） | Null | — |
| 5 | シリンジポンプの管理 | 0:なし / 1:あり | — |
| 6 | 輸血や血液製剤の管理 | 0:なし / 1:あり | — |
| 7 | 専門的な治療・処置（⑤⑩⑪のみ） | 3桁の数字 | 下記参照 |
| 8 | 救急搬送後の入院 | 0:なし / 1:あり | — |

**ペイロード7の3桁ビット列パースルール**

ペイロード7は `⑤放射線治療` / `⑩ドレナージの管理` / `⑪無菌治療室での治療` の3項目を、左から順に1桁ずつで表現した3桁の数字列。

```
桁の対応：
  1桁目（左）= ⑤ 放射線治療
  2桁目（中）= ⑩ ドレナージの管理
  3桁目（右）= ⑪ 無菌治療室での治療

例：
  "100" → ⑤のみあり
  "010" → ⑩のみあり
  "011" → ⑩・⑪あり
  "000" → 全てなし
```

> A12の他の子項目（①②③④⑥⑦⑧⑨）はEFファイルから判定するためHファイルには記録されない。

### 3-7. TAR0010 判定対象フラグのルール

| 値 | 内容 | 集計への影響 |
|---|---|---|
| 0 | 重症度、医療・看護必要度判定対象 | **集計対象** |
| 1 | 短期滞在手術等基本料算定症例 | 一般病棟は集計対象（0扱い）、それ以外は除外 |
| 2 | 年齢が15歳未満 | 除外 |
| 3 | 産科の患者 | 除外 |
| 4 | 外泊日（0時から24時の間の外泊） | 除外 |
| 5 | 退院日（入院した日に退院した場合は除く） | 除外 |

**優先順位ルール：** 複数条件に該当する場合は値の小さいものを優先する。  
**例外：** 一般病棟において15歳未満の短期滞在手術等基本料算定患者は「1」ではなく「2」を選択する。

> 外泊日・退院日はレコード自体を作成しない（そもそも作成不要であり、TAR0010で値を立てる運用ではない）。

## 4. EFファイル取込ルール

### 4-1. ファイル仕様

| 項目 | 仕様 |
|---|---|
| 形式 | タブ区切りテキスト（TSV） |
| 文字コード | Shift-JIS |
| 前ゼロ | ○印フィールドは前ゼロ必須 |

### 4-2. 看護必要度判定に使用するフィールド

全31フィールドのうち、看護必要度判定に使用するフィールドは以下の通り。

| フィールド | DE番号 | 内容 | 用途 |
|---|---|---|---|
| データ識別番号 | EF-2 | 患者識別（Hファイルとの突合キー） | 患者紐付け |
| 退院年月日 | EF-3 | yyyymmdd（在院中は`00000000`） | 在院期間の確認 |
| 入院年月日 | EF-4 | yyyymmdd | 在院期間の確認 |
| データ区分 | EF-5 | 診療識別コード | 診療種別の絞り込み |
| レセプト電算コード | EF-9 | 9桁コード | **A項目・C項目の判定キー** |
| 行為明細区分情報 | EF-17 | 12桁（包括区分等） | 包括項目の扱い判断（下記参照） |
| 実施年月日 | EF-24 | yyyymmdd（Hファイルとの突合キー） | 日次突合・有効期間起算 |
| 病棟コード | EF-28 | 病院独自コード（Hファイルとの突合キー） | 病棟紐付け |
| 病棟区分 | EF-29 | `0`：一般 / `1`：一般以外 / `2`：外来 | 一般病棟の絞り込み |
| 入外区分 | EF-30 | `0`：入院 / `1`：外来 | 入院レコードの絞り込み |

### 4-3. 統合キー（Hファイルとの突合）

| キー | EFファイル（DE番号） | Hファイル | 備考 |
|---|---|---|---|
| 患者識別 | EF-2 データ識別番号 | データ識別番号 | 様式1と一致 |
| 実施日 | EF-24 実施年月日 | 実施年月日 | 日次で突合 |
| 病棟 | EF-28 病棟コード | 病棟コード | 実施日ごとにセット |

### 4-4. レコード絞り込み条件

EFファイルから看護必要度判定に使用するレコードを抽出する際の条件。

| 条件 | フィールド | 値 |
|---|---|---|
| 入院レコードのみ | EF-30 入外区分 | `0`（入院） |
| 一般病棟のみ（一般病棟評価票の場合） | EF-29 病棟区分 | `0`（一般） |

### 4-5. 行為明細区分情報（EF-17）の扱い

EF-17の2桁目（入院料包括項目区分）に注意が必要。

| 2桁目の値 | 意味 | 看護必要度判定での扱い |
|---|---|---|
| `0` | 通常の診療項目 | **判定に使用する** |
| `1` | DPC特定入院料等に包括される項目 | **判定に使用する**（実施された行為として扱う） |
| `2` | その他の入院料に包括される項目 | **判定に使用する**（同上） |

> **根拠：** 看護必要度は「実際に実施されたか否か」で評価するため、点数算定上の包括・非包括に関わらず実施されたレセプト電算コードを判定対象とする。🔲（要最終確認）

### 4-6. A項目・C項目のレセプト電算コード対応表

🔲 **コード対応表は厚労省通知（別表）より取得後に追記**

現時点で確定している判定対象フィールドの構造のみ記録する。

**判定ロジックの基本方針：**

- 対象患者・対象日のEFレコードを抽出し、レセプト電算コード（EF-9）がコード対応表に存在するか照合する
- 該当コードが存在すれば当該A項目またはC項目を「あり（1点）」と判定する
- C項目は実施日から有効期間（項目ごとに異なる）内の全日を「あり」とするため、実施日を起点にした有効期間展開処理が必要

**有効期間展開の考え方：**

```
例：開腹手術（C3）有効期間6日間
  手術実施日：2025-10-05

  → 有効期間内として「あり」とする日：
     🔲 手術日当日から6日間（10/05〜10/10）か
        手術翌日から6日間（10/06〜10/11）かは要確認

  → daily_nursing_evaluation の各日レコードに
     c_score = 1 をセットする処理が必要
```

**データ区分（EF-5）と看護必要度項目の対応（想定）：**

| データ区分 | 診療種別 | 関連するA項目・C項目（想定） |
|---|---|---|
| 21〜27（内服・外用・調剤等） | 薬剤系 | A12-①〜④⑥（抗悪性腫瘍剤・麻薬・免疫抑制剤等） |
| 31〜33（注射） | 注射系 | A12-⑦⑧⑨（昇圧剤・抗不整脈剤・抗血栓塞栓薬）、A4（注射薬剤3種類以上）🔲 |
| 40（処置） | 処置系 | A1・A3・A6（創傷処置・呼吸ケア・シリンジポンプ等）🔲 |
| 50（手術） | 手術系 | C1〜C6・C9（開頭・開胸・開腹・骨・腹腔鏡・全身麻酔・別に定める手術） |
| 54（麻酔） | 麻酔系 | C6（全身麻酔・脊椎麻酔の手術）🔲 |
| 60（検査・病理） | 検査系 | C8（別に定める検査）、C7-①②③（救命等に係る内科的治療）🔲 |

> 🔲 上記の対応はあくまで想定。実際のコード対応表が確定次第、各項目の具体的なレセプト電算コード一覧に置き換える。

## 6. 施設基準充足判定ルール

詳細なマスタデータは参照用ドキュメント `admission_criteria_master_v0.1.md` を参照すること。

### 6-0. プロトタイプ実装スコープ

> **⚠️ プロトタイプでは以下の2入院料のみを実装対象とする。**  
> それ以外の入院料・加算はダミー表示（未実装）とし、将来バージョンで順次対応する。

| 優先度 | 入院料 | 判定パターン |
|---|---|---|
| ✅ 実装 | 一般病棟入院基本料（急性期一般入院料1〜6、地域一般入院料1） | P1・P2 |
| ✅ 実装 | 地域包括ケア病棟入院料1〜4 | P3 |
| 🔲 ダミー | 上記以外の全入院料・加算 | — |

### 6-1. 判定の基本構造

1. 対象期間（月単位）の全評価日について、TAR0010=0（判定対象）の患者・日のみを集計母数とする
2. 各患者・日について「基準を満たす患者」に該当するかを判定する
3. 「基準を満たす患者の割合（%）」を算出し、入院料・加算ごとの閾値と比較する
4. 入院基本料の施設基準と加算の施設基準は**独立して判定する**（一方が他方に従属しない）

### 6-2. 判定パターン一覧

判定条件式は以下の7パターンに集約される。実装時はこのパターンコードを参照する。

| パターンID | 条件式 | 主な該当入院料・加算 |
|---|---|---|
| `P1` | A≥3 **または** C≥1 | 急性期一般1（基準①）、特定機能7対1・専門7対1（基準①） |
| `P2` | (A≥2 **かつ** B≥3) **または** A≥3 **または** C≥1 | 急性期一般2〜5、10対1系、脳卒中CU、地域包括医療（基準①）、急性期患者支援体制加算、看護補助加算 |
| `P3` | A≥1 **または** C≥1 | 特定一般※7、地域包括ケア、総合入院体制加算 |
| `P4` | A≥2 | 特定集中治療室管理料、救命救急2・4 |
| `P5` | HCU独自項目（蘇生術・中心静脈圧測定・人工呼吸器の管理・輸血や血液製剤の管理・肺動脈圧測定・特殊な治療法等）のいずれかに該当（基準①）、**または** A項目のいずれかに該当（基準②） | HCU、救命救急1・3 |
| `P6` | B項目の「診療・療養上の指示が通じる」または「危険行動」のいずれかに該当 | 看護職員夜間配置加算（地域包括ケア） |
| `P7` | 特殊な治療法等（A11）に該当 | 重症患者対応体制強化加算 |
