## 1. システム概要

DPCデータ（HファイルおよびEFファイル）を取り込み、看護必要度を日次で集計・分析することで、入院基本料・加算の施設基準充足状況を確認するツール。

### 処理の全体フロー

```
【入力】
  Hファイル（タブ区切りテキスト）
  EFファイル（タブ区切りテキスト）
        ↓
【取込・パース】
  ヘッダ部の解析（施設コード・病棟コード・患者識別・日付）
  ペイロード部の解析（コード・バージョンで項目を特定）
        ↓
【統合処理】
  Hファイル（手動評価項目）× EFファイル（レセプト電算コード）を
  患者ID × 実施日 をキーに突合・統合
        ↓
【評価票種別の決定】
  病棟コード・入院料種別から使用する評価票を特定
        ↓
【スコア算出】
  評価票種別に応じたA・B・Cスコアを計算
  判定対象フラグ（TAR0010）により除外患者を除く
        ↓
【施設基準充足判定】
  入院基本料・加算ごとの閾値と比較して充足状況を判定
        ↓
【出力】
  集計レポート・充足状況ダッシュボード
```

## 2. データソース設計

### 基本原則

**「Hファイルには手動評価が必要な項目のみ記録される」**

看護必要度の評価方式（ⅠまたはⅡ）によって、各項目のデータ取得元が異なる。

| 評価方式 | データ取得元 | 対象項目 |
|---|---|---|
| 必要度**Ⅰ** | Hファイル（看護師による手動評価） | レセプト電算コードとの対応が困難な項目 |
| 必要度**Ⅱ** | EFファイル（レセプト電算コードから自動判定） | レセプト電算コードで代替可能な項目 |

### 評価票種別 × データソース対応表

| 評価票種別 | A項目ソース | B項目ソース | C項目ソース |
|---|---|---|---|
| 一般病棟Ⅰ（GENERAL_I） | Hファイル（ASS0013）＋EFファイル（一部） | Hファイル（ASS0021） | EFファイル |
| 一般病棟Ⅱ（GENERAL_II） | EFファイル | Hファイル（ASS0021） | EFファイル |
| ICU（ICU） | EFファイル | Hファイル（ASS0051） | 対象外 |
| HCU（HCU） | Hファイル（ASS0062）＋EFファイル（一部） | Hファイル（ASS0071） | 対象外 |
| リハ（REHABILITATION） | 🔲 要確認 | 🔲 要確認 | 対象外 |

> **C項目は評価票種別に関わらず常にEFファイルから取得する。**

| 8 | 救急搬送後の入院 | 0:なし / 1:あり | — |

**ペイロード7の3桁ビット列パースルール**

ペイロード7は `⑤放射線治療` / `⑩ドレナージの管理` / `⑪無菌治療室での治療` の3項目を、左から順に1桁ずつで表現した3桁の数字列。

```
桁の対応：
  1桁目（左）= ⑤ 放射線治療
  2桁目（中）= ⑩ ドレナージの管理
  3桁目（右）= ⑪ 無菌治療室での治療

例：
  "100" → ⑤のみあり
  "010" → ⑩のみあり
  "011" → ⑩・⑪あり
  "000" → 全てなし
```

> A12の他の子項目（①②③④⑥⑦⑧⑨）はEFファイルから判定するためHファイルには記録されない。

### 3-7. TAR0010 判定対象フラグのルール

| 値 | 内容 | 集計への影響 |
|---|---|---|
| 0 | 重症度、医療・看護必要度判定対象 | **集計対象** |
| 1 | 短期滞在手術等基本料算定症例 | 一般病棟は集計対象（0扱い）、それ以外は除外 |
| 2 | 年齢が15歳未満 | 除外 |
| 3 | 産科の患者 | 除外 |
| 4 | 外泊日（0時から24時の間の外泊） | 除外 |
| 5 | 退院日（入院した日に退院した場合は除く） | 除外 |

**優先順位ルール：** 複数条件に該当する場合は値の小さいものを優先する。  
**例外：** 一般病棟において15歳未満の短期滞在手術等基本料算定患者は「1」ではなく「2」を選択する。

> 外泊日・退院日はレコード自体を作成しない（そもそも作成不要であり、TAR0010で値を立てる運用ではない）。

## 4. EFファイル取込ルール

### 4-1. ファイル仕様

| 項目 | 仕様 |
|---|---|
| 形式 | タブ区切りテキスト（TSV） |
| 文字コード | Shift-JIS |
| 前ゼロ | ○印フィールドは前ゼロ必須 |

### 4-2. 看護必要度判定に使用するフィールド

全31フィールドのうち、看護必要度判定に使用するフィールドは以下の通り。

| フィールド | DE番号 | 内容 | 用途 |
|---|---|---|---|
| データ識別番号 | EF-2 | 患者識別（Hファイルとの突合キー） | 患者紐付け |
| 退院年月日 | EF-3 | yyyymmdd（在院中は`00000000`） | 在院期間の確認 |
| 入院年月日 | EF-4 | yyyymmdd | 在院期間の確認 |
| データ区分 | EF-5 | 診療識別コード | 診療種別の絞り込み |
| レセプト電算コード | EF-9 | 9桁コード | **A項目・C項目の判定キー** |
| 行為明細区分情報 | EF-17 | 12桁（包括区分等） | 包括項目の扱い判断（下記参照） |
| 実施年月日 | EF-24 | yyyymmdd（Hファイルとの突合キー） | 日次突合・有効期間起算 |
| 病棟コード | EF-28 | 病院独自コード（Hファイルとの突合キー） | 病棟紐付け |
| 病棟区分 | EF-29 | `0`：一般 / `1`：一般以外 / `2`：外来 | 一般病棟の絞り込み |
| 入外区分 | EF-30 | `0`：入院 / `1`：外来 | 入院レコードの絞り込み |

### 4-3. 統合キー（Hファイルとの突合）

| キー | EFファイル（DE番号） | Hファイル | 備考 |
|---|---|---|---|
| 患者識別 | EF-2 データ識別番号 | データ識別番号 | 様式1と一致 |
| 実施日 | EF-24 実施年月日 | 実施年月日 | 日次で突合 |
| 病棟 | EF-28 病棟コード | 病棟コード | 実施日ごとにセット |

### 4-4. レコード絞り込み条件

EFファイルから看護必要度判定に使用するレコードを抽出する際の条件。

| 条件 | フィールド | 値 |
|---|---|---|
| 入院レコードのみ | EF-30 入外区分 | `0`（入院） |
| 一般病棟のみ（一般病棟評価票の場合） | EF-29 病棟区分 | `0`（一般） |

### 4-5. 行為明細区分情報（EF-17）の扱い

EF-17の2桁目（入院料包括項目区分）に注意が必要。

| 2桁目の値 | 意味 | 看護必要度判定での扱い |
|---|---|---|
| `0` | 通常の診療項目 | **判定に使用する** |
| `1` | DPC特定入院料等に包括される項目 | **判定に使用する**（実施された行為として扱う） |
| `2` | その他の入院料に包括される項目 | **判定に使用する**（同上） |

> **根拠：** 看護必要度は「実際に実施されたか否か」で評価するため、点数算定上の包括・非包括に関わらず実施されたレセプト電算コードを判定対象とする。🔲（要最終確認）

### 4-6. A項目・C項目のレセプト電算コード対応表

**参照ファイル：** `general_ward_ac_item_code_master_v0.2.tsv`  
**件数：** 2,591件（A項目：1,059件 / C項目：1,532件）  
**根拠：** 厚労省通知別表（「重症度、医療・看護必要度」に係る評価票 評価の手引き）

#### スキーマ定義

| フィールド | 型 | 内容 |
|---|---|---|

#### level1_only フラグの使用方法

判定処理において評価票種別に応じてコードを絞り込む。

```
看護必要度Ⅱの場合：level1_only = 0 のコードのみ使用
看護必要度Ⅰの場合：level1_only = 0・1 両方のコードを使用
```

**管理方針：** 診療報酬改定ごとにマスタファイルを更新する。業務ルール定義書本体のコード変更は不要。

#### 判定ロジックの基本方針

- 対象患者・対象日のEFレコードを抽出し、EF-9（レセプト電算コード）がマスタに存在するか照合する
- 該当コードが存在すれば当該A項目またはC項目を「あり（1点）」と判定する
- C項目は実施日から有効期間（項目ごとに異なる）内の全日を「あり」とするため、実施日を起点にした有効期間展開処理が必要

**有効期間展開の考え方：**

```
例：開腹手術（GEN_C17）有効期間6日間
  手術実施日：2025-10-05

  → 有効期間内として「あり」とする日：
     🔲 手術日当日から6日間（10/05〜10/10）か
        手術翌日から6日間（10/06〜10/11）かは要確認

  → daily_nursing_evaluation の各日レコードに
     c_score = 1 をセットする処理が必要
```

### 4-7. GEN_A3（注射薬剤3種類以上の管理）判定ロジック

#### 対象評価票種別

| 評価票種別 | 正規項目ID | データソース | 備考 |
|---|---|---|---|
| GENERAL_I（一般病棟Ⅰ） | GEN_A3 | EFファイル | 本セクションのルールを適用 |
| GENERAL_II（一般病棟Ⅱ） | GEN_A3 | EFファイル | 同上 |
| HCU | HCU_A4 | Hファイル（ASS0062 ペイロード4） | EFファイルは使用しない |

#### 判定ロジック（EFファイルを使用する場合）

```
【ステップ1】レコード抽出
  対象患者・対象日のEFレコードのうち、以下の条件をすべて満たすものを抽出する
    - EF-30（入外区分） = 0（入院）
    - EF-29（病棟区分） = 0（一般）※一般病棟の場合
    - EF-5（データ区分） が 30番台（31〜39）※注射系

【ステップ2】除外薬剤の除去
  ステップ1で抽出したレコードのうち、EF-9（レセプト電算コード）が
  「A4除外薬剤マスタ」に含まれるものを除外する

【ステップ3】種類数カウント
  残ったレコードの EF-9（レセプト電算コード）のユニーク値をカウントする
  ※同一コードが同日に複数レコードあっても1種類として扱う

【ステップ4】判定
  カウント結果 ≥ 3 → GEN_A3 = 1（あり）
  カウント結果 ＜ 3 → GEN_A3 = 0（なし）
```

#### 補足事項

- EF-5（データ区分）の30番台は、厚労省DPC仕様における注射系（31：皮下・筋肉内注射、32：静脈内注射、33：その他の注射）を対象とする
- 除外薬剤は「注射薬剤3種類以上の管理」の対象外とされる薬剤（生理食塩液・輸液・ヘパリン等の溶解補助剤・電解質補正液など）
- 除外薬剤マスタは厚労省通知別表に基づき管理し、診療報酬改定時にマスタ更新で対応する
